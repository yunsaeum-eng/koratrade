"저장에 실패했습니다. 다시 시도해주세요." and does nothing.

Steps to fix:
1. Open src/app/onboarding/page.tsx
2. Find the handleFinish (or handleSubmit) function
3. Add console.log at every step to trace where it fails
4. Check that saveProfile() is being called with the correct columns that exist in Supabase profiles table
   - Allowed columns: id, name, avatar, avatar_bg, goal, english_level, industry, motivation, xp, level, streak_days, last_login_date, ui_language

5. Make sure the function is async and properly awaits saveProfile()

   - Wrap ui_language in a try/catch or use optional inclusion
7. After fixing, the submit button should successfully save and redirect to /commute

FIX 2 — Full English UI when English is selected
--------------------------------------------------
Problem: When user selects "English" during onboarding, subsequent screens (character selection,
avatar gender, etc.) still show Korean text. The entire onboarding flow and main app UI must
switch to English.

Steps to fix:

2A. Onboarding pages — translate all steps to English when ui_language === 'english':
  - Character/avatar selection: "여성 캐릭터" → "Female character", "남성 캐릭터" → "Male character"
  - Avatar color labels: translate all Korean labels
  - Goal selection: already bilingual, keep as is
  - English level: "초급" → "Beginner", "중급" → "Intermediate", "고급" → "Advanced"
  - Industry input placeholder: translate
  - Learning reason placeholder: translate
  - All button text: "다음" → "Next", "이전" → "Back", "완료" → "Done", "저장하기" → "Save"
  - Step indicator labels: translate

2B. Detect ui_language during onboarding:
  - The user selects language BEFORE the character/avatar steps
  - Store the selected language in component state immediately when selected
  - Use this state to conditionally render Korean or English labels for ALL subsequent steps in onboarding
  - Do NOT wait for Supabase to load — use local state

2C. Main app UI — apply ui_language from profile:
  - In src/hooks/useLanguage.ts (or create if missing), load ui_language from AuthContext profile

    - src/components/profile/UserProfileModal.tsx (all labels and buttons)
