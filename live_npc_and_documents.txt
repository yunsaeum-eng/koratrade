Implement two critical systems. These are the most important features of this app.

SYSTEM 1 — CLAUDE API LIVE NPC RESPONSES

Replace ALL pre-scripted NPC messages with real-time Claude API responses. This is the core fix.

How it works:
Every time the user sends a message, call Claude API with a carefully constructed system prompt that includes:
- Who the NPC is (personality, speech style, relationship level)
- Current episode context (EP01 = first day, what has happened so far)
- Full conversation history (last 10-15 messages for context)
- User profile (name, goal, language setting)
- What the NPC is allowed to talk about in this episode
- Any active story events or conflicts

The NPC then responds naturally to whatever the user actually said.

System prompt structure for each NPC call:

For James Park (Korean mode, EP01):
"You are James Park, a 3rd year employee at KoraTrade trading company. You are friendly, casual, and funny. You speak in casual Korean (반말 mixed with some 요 endings like a friendly senior colleague). Today is the user's very first day at work. You are showing them around and welcoming them. You can only talk about: first day topics, office layout, team introductions, basic office culture tips, warning about Park manager in a funny way. Do NOT mention buyers, KOTRA, cold emails, or any real work tasks yet. The user just said: [USER_MESSAGE]. Respond naturally to what they actually said. Keep response to 2-3 sentences. Stay completely in character."

For Sarah Kim (Korean mode, EP01):
"You are Sarah Kim, team manager at KoraTrade with 10 years experience. You speak formally but warmly in Korean (존댓말). Today is the user's first day. You gave them a brief welcome and assigned them to read the company profile document. If they ask where to find it, tell them it is in the shared folder on the company drive, James can show them. Respond naturally to what the user actually said: [USER_MESSAGE]. Keep response to 1-2 sentences. Professional tone."

For Klaus Muller (English mode, buyer):
"You are Klaus Muller, purchasing manager at GermanParts in Germany. You are direct, formal, and efficient. You write in professional English. You prefer concise communication. Respond to what the user wrote: [USER_MESSAGE]. If their email or message is unprofessional, respond with confusion or request for clarification. Keep response to 2-3 sentences."

API call implementation:
- Model: claude-haiku-4-5 (fast and cheap for chat)
- Max tokens: 150 (keep responses short and natural)
- Include last 10 messages as conversation history
- Stream the response for real-time typing feel
- Show typing indicator while waiting for response
- If API fails, show a generic "잠시 후 다시 시도해주세요" message

Language handling:
- Korean mode: all NPCs respond in Korean (except Klaus who is always English)
- English mode: all NPCs respond in English
- Pass language setting in system prompt

SYSTEM 2 — DYNAMIC DOCUMENT GENERATION

When the context requires a document (user asks for company profile, NPC mentions sending a file, user needs to review a contract etc), automatically generate and display that document.

How to detect when a document is needed:
- User asks "회사 소개 자료 어디있나요" or "where is the company profile"
- NPC says "파일 첨부해드릴게요" or "please find attached"
- Episode context requires reviewing a specific document
- User needs to fill out a form or write a document

When document is needed, call Claude API with a document generation prompt:

Example — Company Profile Document:
"Generate a realistic Korean trading company profile for KoraTrade Inc. Include: company overview (founded 2008, Seoul headquarters, 45 employees, specializes in industrial parts export to Europe and North America), key products (precision automotive components, electronic parts, industrial machinery parts), major markets (Germany, USA, France, Singapore), certifications (ISO 9001, ISO 14001), key achievements, contact information. Format as a clean professional HTML document with KoraTrade branding (gold color #8a6530). Keep it realistic and detailed enough to feel like a real company document."

Display the document as:
- A modal popup that opens over the chat
- Looks like a real document viewer with title bar showing filename
- Has a close button and a "업무 노트에 저장" button
- Document content generated by Claude API in real time
- Nicely formatted with headers, sections, company logo emoji

Types of documents to auto-generate based on context:
1. Company profile (회사 소개서) - EP01 when user asks
2. Product catalogue (제품 카탈로그) - EP04-05 when preparing cold email
3. Buyer research report (바이어 분석 보고서) - EP03-04 when researching GermanParts
4. Email draft (이메일 초안) - when user needs help drafting
5. Meeting agenda (회의 안건) - when team meeting is called
6. Price quotation (견적서) - Season 4 negotiation phase

Document trigger logic:
- Analyze each user message and NPC response for document-related keywords
- If detected, generate appropriate document and show as attachment bubble in chat
- Attachment bubble shows: file icon, document name, "열어보기" button
- Clicking opens the document modal

This makes the chat feel like a real work environment where documents, files, and resources appear naturally as part of the workflow.
